# 自选股短线风险收益排行榜

## 项目规划与立项背书 v1.0

> 文档状态：需求已确认，可进入技术设计与开发阶段  
> 目标用户：正在学习商科、希望理解市场分析逻辑的学生  
> 使用边界：仅作研究和学习，不连接证券账户，不自动下单

## 1. 项目背景

现有“家庭股票自动看板”已经能够维护自选股、展示日 K 线、成交量和均线，并在每天收盘后自动更新。新阶段不再只是“展示行情”，而是要回答一个更具体的问题：

**在自选股中，哪些股票未来 3 至 5 个交易日更值得研究，同时其风险处于什么水平？**

项目将从历史行情中提取技术状态、行业强弱、量价关系和相似走势，用统一口径计算风险与收益统计，形成每日排行榜。系统不把统计结果描述成确定预测，也不使用“保证上涨”“必买”等表述。

## 2. 已确认需求

| 维度 | 确认内容 |
|---|---|
| 分析范围 | 扫描 `watchlist.csv` 中的自选股票与 ETF |
| 观察周期 | 未来 3 个交易日和未来 5 个交易日 |
| 核心输出 | 风险分数、预期收益百分比、上涨样本比例、排行榜 |
| 市场规律 | 行业走向、技术指标、历史相似走势、K 线与量价结构 |
| 更新频率 | 每个交易日北京时间 17:30 |
| 风险分类 | 绿色=低风险、黄色=中等风险、红色=高风险；必须同时显示文字 |
| 解释程度 | 展示简要加分和扣分原因，不堆砌公式 |
| 页面受众 | 商科学生，结论清楚，同时能展开查看依据 |
| 交易边界 | 只提供研究分析，不自动交易，不连接证券账户 |

## 3. 产品定位

这不是荐股软件，而是一套“自选股研究排序工具”。它的价值在于：

1. 把多只自选股放到同一套规则下比较，减少只凭感觉选股。
2. 同时展示收益统计和风险分数，避免只看上涨空间。
3. 每个结论都能追溯到行业、趋势、量价和历史样本。
4. 每天固定时间自动更新，形成连续、可复盘的研究记录。
5. 为商科学生提供可理解的量化分析案例，而不是黑箱答案。

## 4. 页面最终应回答的问题

用户打开页面后，应能在第一屏快速得到：

- 今天自选股的风险收益排行榜。
- 每只股票未来 3 日、5 日的历史统计收益百分比。
- 风险分数及低/中/高风险标签。
- 为什么加分、为什么扣分。
- 所属行业近期是强于还是弱于大盘。
- 历史相似走势有多少个样本，统计可信度如何。

## 5. 排行规则

系统提供三个排行榜视图：

| 视图 | 默认排序 | 用途 |
|---|---|---|
| 风险收益榜 | 风险调整后评分从高到低 | 默认视图，平衡收益与风险 |
| 收益预估榜 | 5 日统计预期收益从高到低 | 观察短线收益潜力 |
| 低风险榜 | 风险分数从低到高 | 优先研究波动和回撤较低的标的 |

风险分数统一采用 `0–100`：分数越高，风险越大。

- `0–35`：绿色，低风险观察。
- `36–65`：黄色，谨慎观察。
- `66–100`：红色，高风险。

预期收益必须标注为“基于历史相似样本的统计值”，不得称为确定收益。

## 6. 分析框架

### 6.1 技术状态

建议纳入 MA5/10/20/60、RSI14、MACD、ATR14、布林带位置、5/20 日动量、量比、近 20 日波动和近 60 日最大回撤。

### 6.2 行业走向

使用 `watchlist.csv` 的 `category` 字段匹配行业板块，计算行业近 5/20/60 日收益、相对大盘强度和行业趋势分。ETF 或无法匹配行业的标的，使用对应宽基指数作为参照，并明确标记为“指数参照”。

### 6.3 历史相似走势

使用至少 5 年日线建立历史样本。每个历史时点只能使用该时点之前的数据，寻找与当前技术、量价和行业状态相近的片段，再统计其后 3 日和 5 日的实际涨跌。必须防止未来数据泄漏。

### 6.4 K 线与量价

保留现有 K 线图，并增加对当前结构的简短描述，例如“价格站上 20 日均线”“上涨伴随放量”“跌破近期支撑”。不要求自动识别复杂主观形态。

## 7. 技术架构背书

采用现有的 GitHub Actions + GitHub Pages 静态架构：

```text
交易日 17:30 GitHub Actions
        |
        v
DeepSeek 后端脚本抓取数据、计算指标、回测相似样本
        |
        v
生成 ranking.json 与 analysis/{code}.json
        |
        v
Gemini 前端读取 JSON，展示排行榜、解释和 K 线
        |
        v
GitHub Pages 自动发布
```

选择该架构的原因：

- 家庭电脑关机不影响更新。
- 不需要付费数据库或常驻服务器。
- 每日收盘分析不需要实时接口，静态 JSON 更稳定。
- 后端与前端通过固定数据合同解耦，适合分别交给两个模型开发。
- GitHub 保留每次数据变化，便于复盘与追责。

## 8. 分工

### DeepSeek

负责数据抓取、行业映射、技术指标、历史相似样本、风险与收益计算、排行榜 JSON、自动任务、测试与日志。不得修改前端字段名称。

### Gemini 3.5 Flash

负责排行榜、筛选排序、三色风险、个股分析面板、K 线整合、移动端适配和学习型解释。不得在浏览器中重新计算核心评分。

### 共同约束

双方必须遵守《共享数据合同》。任何字段变更先修改合同，再同步修改两端，禁止各自猜测字段。

## 9. 开发里程碑

1. 后端用固定测试数据生成符合合同的 JSON。
2. 前端用示例 JSON 完成所有界面状态。
3. 接入真实自选股与行业数据。
4. 完成无未来数据泄漏的历史相似样本统计。
5. 在本地和 GitHub Actions 运行测试。
6. 灰度运行 10 个交易日，记录预测值与实际结果。
7. 根据复盘结果调整权重，之后再作为正式版发布。

## 10. 验收标准

- 每个交易日 17:30 后自动更新，非交易日不伪造新结果。
- 自选股全部出现在排行榜，单只失败不会拖垮全局。
- 风险分、3 日/5 日统计收益、样本量、更新时间均可见。
- 三种风险状态同时有颜色和文字，不依赖颜色辨认。
- 任一评分都能展示主要加分/扣分原因。
- 历史相似计算通过防未来数据泄漏测试。
- 手机与电脑均无文字重叠、横向溢出或无法点击的问题。
- 页面明确展示研究用途声明，不出现自动交易功能。

## 11. 主要风险与控制

| 风险 | 控制措施 |
|---|---|
| 数据源临时不可用 | 重试、备用源、保留上次成功数据并标记陈旧 |
| 样本过少导致结论不稳 | 显示样本量与置信等级，低于阈值不输出收益估计 |
| 模型过拟合 | 初版使用透明规则和 KNN，不直接使用黑箱深度学习 |
| 未来数据泄漏 | 按时间切分测试，历史时点只能读取当时可用数据 |
| 把统计当成投资建议 | 固定风险声明，不使用“买入/卖出/保证”等词语 |
| 行业映射失败 | 使用指数参照并记录原因，不静默填充虚假行业 |

## 12. 项目成功定义

项目成功不等于每天猜对涨跌，而是满足三点：排行榜口径稳定、风险收益依据可解释、预测结果可以长期记录并复盘。只有经过足够交易日验证后，才讨论是否调整因子和权重。
