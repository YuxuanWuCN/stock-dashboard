# Gemini 3.5 Flash 前端代码需求

## 1. 任务目标

在现有“家庭股票自动看板”中增加“自选股短线风险收益排行榜”。前端读取后端生成的静态 JSON，不在浏览器中重新计算核心评分，不调用大语言模型，不连接证券账户。

页面面向正在学习商科的学生：结论要快速可读，依据要能展开研究，不能做成营销页或复杂交易终端。

## 2. 技术约束

- 延续现有纯静态 HTML + CSS + JavaScript + ECharts。
- 不引入 React、Vue、构建工具或新的后端。
- 修改现有 `docs/index.html`、`docs/assets/app.js`、`docs/assets/style.css`。
- 严格读取《04-前后端共享数据合同.md》规定的字段。
- 保留现有自选股列表、K 线、单股查询和编辑列表功能。

## 3. 页面结构

### 3.1 排行榜区域

在现有查询栏下方、股票详情布局上方增加排行榜。默认第一屏必须能看到排行榜前几名。

使用三个标签页或分段控制：

1. `风险收益榜`：按 `risk_adjusted_score` 降序。
2. `收益预估榜`：按 `forecast.return_5d_pct` 降序，`null` 排最后。
3. `低风险榜`：按 `risk.score` 升序。

桌面端表格列：

| 列 | 内容 |
|---|---|
| 排名 | 当前视图排序名次 |
| 股票 | 名称、代码、行业 |
| 风险收益分 | `risk_adjusted_score` |
| 3 日统计收益 | `forecast.return_3d_pct` |
| 5 日统计收益 | `forecast.return_5d_pct` |
| 上涨样本比例 | 5 日比例与样本数 |
| 风险 | 风险分数、文字标签和颜色 |
| 主要依据 | 第一条加分/扣分原因 |

移动端改为紧凑行或单层列表，不允许在卡片内部再嵌套卡片。

### 3.2 三色风险

风险颜色使用交通灯语义，并始终附带文字：

- 低风险：绿色 `#15803d`，显示“低风险”。
- 中等风险：黄色/琥珀色 `#b45309`，显示“中等风险”。
- 高风险：红色 `#b91c1c`，显示“高风险”。

不能只用颜色表达风险。收益涨跌仍按 A 股习惯显示正值红、负值绿，但风险标签必须有边框、文字和固定位置，避免混淆。

### 3.3 个股分析区

点击排行榜中的一只股票后，复用现有 K 线详情区域并增加：

- 股票名称、代码、行业、交易日和数据状态。
- 风险收益分、风险分、3 日/5 日统计收益。
- 当前技术状态和行业相对强弱。
- 最多 5 条主要依据，分为正面、提醒、负面。
- 历史相似样本：样本数、上涨比例、平均/中位收益、最好/最差结果。
- 现有 K 线与成交量图。
- 固定免责声明。

使用清晰的小标题和指标网格，不使用超大字体，不将每个数字都做成装饰性卡片。

## 4. 学习型表达

页面可以解释领域概念，但不要写使用教程。建议：

- 指标名称旁提供简短提示，例如“风险分越高表示历史波动和回撤越大”。
- “评分依据”直接展示真实数据，如“行业 20 日跑赢大盘 3.1%”。
- 低置信结果显示“历史样本较少”，不要显示假精确的小数。
- 不使用“买入”“卖出”“稳赚”“明天必涨”等词语。
- 建议状态只使用“低风险观察”“谨慎观察”“高风险”。

## 5. 状态设计

必须实现：

- 首次加载骨架或固定高度加载状态，不能导致布局跳动。
- 数据成功状态。
- 部分股票失败状态。
- 数据陈旧状态，显示最后成功交易日。
- `ranking.json` 不存在或格式错误状态。
- 某只详情文件加载失败状态。
- 预期收益为 `null` 时显示“样本不足”，不能显示 `0.00%`。

## 6. 排序与交互

- 切换排行榜视图后立即本地排序，不重新请求后端。
- 点击列标题可切换升降序，显示清晰的排序图标。
- 可按行业过滤，可搜索代码或名称。
- 点击排行行进入该股分析；再次点击其他股票时不刷新整页。
- 支持键盘焦点和触屏点击区域。
- 所有按钮、表格列和图表在 360px 手机宽度下不得重叠。

## 7. 视觉要求

- 延续现有白底、克制、工作型看板风格。
- 新增区域圆角不超过 8px。
- 不使用渐变球、装饰光斑、营销式大标题或嵌套卡片。
- 指标表格和排行榜尺寸稳定，加载和切换时不得抖动。
- 字符间距为 `0`，不按视口宽度缩放字号。
- 风险颜色、收益颜色、行业标签需有明确层级，页面不能被单一颜色统治。
- 图表沿用 ECharts，禁止手绘重复 SVG 图标。

## 8. 数据读取

启动时并行加载：

```text
data/meta.json
data/summary.json
data/analysis/ranking.json
```

点击个股时加载：

```text
data/kline/{code}.json
data/analysis/{code}.json
```

缓存已加载的详情，避免重复请求。所有字段读取前检查 `null` 和缺失情况。

## 9. 安全与边界

- 使用 `textContent` 或安全 DOM API 展示后端文本，避免直接拼接未转义 HTML。
- 不在前端保存密钥、Token 或用户证券信息。
- 不添加自动下单、交易账户、持仓金额或个性化投资建议。
- 固定展示：`基于历史日线的统计分析，仅用于学习和研究，不构成投资建议或收益保证。`

## 10. 验收标准

- 三个排行榜视图排序正确，`null` 始终排最后。
- 风险分与颜色、文字标签完全一致。
- 正负收益显示正确，不能与风险颜色语义混淆。
- 点击股票后展示对应详情和 K 线。
- 样本不足、陈旧、失败等状态均可见且不破坏布局。
- 360px、768px、1440px 三种宽度无重叠和横向溢出。
- 浏览器控制台无错误。
- 原有自选股、查询、编辑功能正常。

## 11. 给 Gemini 3.5 Flash 的执行指令

请先阅读现有前端代码与共享数据合同，先提交页面结构和拟修改文件列表，确认后再编码。不要重写整个页面，不要引入框架。先使用合同中的示例 JSON 完成所有状态，再接真实文件。完成后必须提供桌面与手机截图、控制台检查结果和一份逐项验收清单。
