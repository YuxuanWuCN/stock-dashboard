# 前后端共享数据合同

## 1. 基本规则

- 合同版本：`2.0`。
- 文件编码：UTF-8。
- 日期：`YYYY-MM-DD`。
- 时间：ISO 8601，必须包含 `+08:00`。
- 百分比字段直接使用百分数，例如 `2.35` 表示 `2.35%`。
- 分数字段范围 `0–100`。
- 无可靠结果必须使用 `null`，禁止用 `0`、`--` 或空字符串冒充。
- 后端输出英文键名；前端负责中文显示。
- 未经双方同步，不得删除、改名或改变字段类型。

## 4. 枚举

### 风险等级

| `level` | 分数 | 中文 |
|---|---:|---|
| `low` | 0–35 | 低风险 |
| `medium` | 36–65 | 中等风险 |
| `high` | 66–100 | 高风险 |

### 原因类型

- `positive`：加分原因。
- `warning`：需要观察。
- `negative`：扣分或风险原因。

### 置信等级

- `low`：样本不足或相似度较低。
- `medium`：样本达到最低要求。
- `high`：样本充分且相似度较高。

### 趋势

- `strong_uptrend`
- `uptrend`
- `range`
- `rebound`
- `downtrend`
- `insufficient`

## 5. 兼容性规则

- 前端遇到未知新增字段时忽略，不报错。
- 后端不得改变已有字段类型。
- 新增可选字段允许，但必须先提升次版本并更新本文档。
- 删除或改名字段属于破坏性变更，必须提升主版本。
- `schema_version` 不兼容时，前端显示“分析数据版本不兼容”，不得继续猜测解析。

## 6. 校验要求

后端必须提供 JSON Schema 或等价校验测试，至少验证：

- 必填字段存在。
- 分数在 `0–100`。
- 风险等级与风险分数一致。
- 百分比是数字或 `null`。
- 排名不重复。
- `succeeded + failed = total`。
- `trade_date` 不晚于北京时间当前日期。
- `stale=true` 时保留最后成功交易日信息。
